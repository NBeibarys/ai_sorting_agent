### What changed

Removed the hardcoded 16-column `OUTPUT_HEADER` (Alchemist sheet column order) from `src/pipeline.py`. The actual source sheet header (read at runtime via `read_sheet_rows`) is passed to `write_tab_data`. Output rows are built by index in source order (full_row copied verbatim), so using the source header aligns columns correctly for any sheet — Alchemist (16 cols), R2B (23 cols), or future sheets — instead of writing R2B data under Alchemist's header and landing values in wrong columns.
